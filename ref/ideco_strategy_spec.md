## 戦略概要: Core-Satellite

### 設計思想

Gary Antonacci の Dual Momentum を日本 iDeCo 向けにアレンジ。
バックテスト結果（旧モメンタム: 年率+2.6%）を踏まえ、以下の問題を解決するために改定。

**旧戦略の問題点**:
1. Goldが高頻度で選ばれ信託報酬（0.895%）が累積
2. MA3>MA6フィルターがV字回復時の再エントリーを遅延
3. TOP_N=3では不振商品も混入しやすい

**新戦略の改善**:
1. 先進国株式（最低コスト0.099%）を Core として常時70%保有しベータを確保
2. MA3>MA6条件を廃止してV字回復を取り逃さない
3. Satellite 1枠に集中してモメンタム判断の精度を上げる
4. Goldの実効信託報酬を×2倍にしてスコアを抑制

### バックテスト比較（2018-10〜2025-02、77ヶ月、総拠出額177万円）

| 戦略 | 最終資産額 | 運用益率 | 年率 | MaxDD |
|------|---:|---:|---:|---:|
| 旧モメンタム | 2,084,531円 | +17.7% | +2.6% | 22.0% |
| **CS固定（Core常時70%・推奨）** | **3,244,257円** | **+83.2%** | **+9.9%** | **21.6%** |
| CS可変（Core SELL時→元本保証） | 2,426,257円 | +37.0% | +5.0% | 10.2% |
| 均等B&H（7本） | 2,544,541円 | +43.7% | +5.8% | 12.0% |
| 先進国株式のみ | 3,331,624円 | +88.1% | +10.3% | 20.3% |
| 全米株式のみ | 3,422,319円 | +93.2% | +10.8% | 19.8% |

> CS固定: CORE_ABS_MOMENTUM=false（Core 常時70%）。先進国株式 B&H の 94% の実績。
> CS可変: CORE_ABS_MOMENTUM=true（Core SELL時→元本保証）。ドローダウンは小さいが長期成長を大きく損なう。

## スコアリングロジック

### 基本スコア計算式

```
スコア = (1M×1 + 3M×2 + 6M×4 + 12M×3) / 10 − 実効信託報酬(%/年)
```

- 6Mウェイトを最重視（中期モメンタムが最も安定）
- 12Mウェイトを6Mより軽く（長期過剰反応を抑制）
- 実効信託報酬: Goldは 信託報酬 × 2.0 でペナルティ

### データ取得元

```
基準価額ページURL例：
https://www.rakuten-sec.co.jp/web/fund/detail/price.html?Code={楽天内コード}&date={YYYY}%20{M}%20{D}
```

スクレイピング対象データ:
- 基準価額（円）
- 各期間騰落率（1ヶ月・3ヶ月・6ヶ月・12ヶ月）
- 信託報酬合計（年率）

## BUY/SELLシグナル判定

### ステップ1: 絶対モメンタムフィルター（ゲート条件）

```python
# 条件①: スコアがプラス
score > 0

# 条件②: 12ヶ月リターンがプラス（絶対モメンタム）
return_12m > 0
```

両方を満たした商品のみをBUY候補とする。

> 条件③（MA3>MA6）は廃止。バックテストで V字回復の取り逃しを引き起こしていたため。

## Core-Satellite ポートフォリオ構築

### Core（固定70%）

```
商品: たわらノーロード先進国株式（JP90C000CMK4）
配分: 月次掛金の70%（常時固定）
理由: 最低コスト(0.099%)・先進国全体をカバー・安定したベータリターン
```

### Satellite（可変30%）

```
BUY候補からスコア上位1本を選定（Coreを除く8商品対象）
→ 月次掛金の残り30%を割当
→ BUY候補ゼロ時: Core に100%（Satellite枠もCore）
```

### 元本保証退避

```
条件: 全BUY候補がゼロ（Core以外は全SELL）
対応: Satellite枠もCoreへ（掛金は100% Core）
     ケースAルールで既存Satellite残高も順次Coreへスイッチング
```

## スイッチングルール

### ケースA（高優先度）

```
条件: 非Core商品のSELLシグナルが2ヶ月連続かつ残高あり
対応: 該当商品の残高全額をCore商品（たわら先進国株式）へスイッチング
```

### ケースB（廃止）

Core-Satlliteモードでは無効化。
Core商品はレガシー保有（楽天全米株式 49%等）と目標配分が乖離するため、
ケースBを適用すると誤ったスイッチングが発動してしまう。

## 月次実行フロー

```
① スクレイピング
   楽天証券の各商品ページから基準価額・騰落率・信託報酬を取得
   対象: iDeCo口座の取扱商品リスト（9商品）

② スコア計算
   実効スコア = 加重モメンタム − 実効信託報酬
   （Gold は信託報酬×2倍で計算）

③ フィルタリング
   条件① score > 0
   条件② 12Mリターン > 0
   → BUY候補リスト生成（Coreを含む全商品対象）

④ 掛金割当決定
   Core: 70%固定（BUY/SELLに関わらず）
   Satellite: 非Core BUY候補の上位1本に30%
   候補なし: Core 100%

⑤ スイッチング判定
   ケースA: 2ヶ月連続SELL の非Core保有 → Core へ全額移動

⑥ Core候補モニタリング
   CORE_CANDIDATESの各商品のスコアとCoreスコアを比較
   候補が連続してCoreを上回った月数を core_monitor_history.json に記録
   CORE_CHANGE_MONTHSヶ月連続で上回った候補は月次レポートに変更提案を表示

⑦ アクション出力
   - 掛金割当変更先と比率
   - スイッチング対象・金額・行き先
   - 判定根拠（スコア一覧）
   - Core候補モニタリング状況（連続月数・変更提案）
```

## Core候補モニタリング

Core商品（たわら先進国株式）は永続的に固定するリスクがあるため、月次で候補商品とスコアを比較して変更タイミングを検出する。

### 候補商品

| 商品名 | ISIN | 信託報酬 | 選定理由 |
|---|---|---:|---|
| 楽天・プラス・S&P500インデックス・ファンド | JP90C000Q2U6 | 0.077% | 超低コスト米国株式 |
| 楽天・プラス・オールカントリー株式インデックス・ファンド | JP90C000Q2W2 | 0.056% | 最安コスト・全世界カバー |

### 判定ルール

```
毎月: 候補スコア > Core スコア → beats_core=true
連続beats_core が CORE_CHANGE_MONTHS（=3）ヶ月以上 → レポートに⚠変更検討を表示
```

### Core変更手順（手動）

1. 月次レポートで「⚠ 変更検討」を確認
2. バックキャスト（`tools/ideco_backcast.py`）でCore比較を確認
3. `config/ideco_products.json` の `CORE_PRODUCT` を変更先のISINに書き換え
4. 楽天証券DCサイトで既存Core残高を新Core商品へスイッチング申請

> **注意**: Core変更は自動では行われない。iDeCoのスイッチング申請ラグ（数営業日）を考慮して手動で実施する。

## 注意事項

- 楽天証券のページ構造は変更される可能性がある → セレクタはconfig化して変更しやすくする
- iDeCoのスイッチングには反映ラグがある（申請から数営業日）→ 月初判定・月中実行を想定
- Core商品（たわら先進国株式）のBUY/SELLシグナルはレポートに表示されるが、Core配分は固定（シグナルに関わらず70%）
- バックテストは手数料・スイッチングコストを含まない簡易試算
